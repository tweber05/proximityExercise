Columns

NOTE: Before getting started with this exercise, make sure to increase the size of your browser window to take up the full screen of your laptop!
The ruler below is marked in 10 pixel increments. Below the ruler are two divs, testDiv1 and testDiv2. Let's make them into columns!! Try the following and see what happens:
  1. Note how large each div currently is by measuring them on the ruler
  2. Set the "box-sizing" property to "border-box" on each div. How does this change the size of the divs?
  3. Float both divs right, what happens?
  4. Float both divs left, what happens?
  5. Resize your browser window such that it is less than 600 pixels wide (use the ruler to measure). What happens? Set your browser window back to the full width of your screen when done.
  6. Float testDiv1 right and testDiv2 left. What happens?
  7. Float both divs to the left or to the right and set the size of testDiv1 to 2000px. What happens?
  8. So far, we have been creating fixed layouts by setting the width of our columns in pixels. The advantage of a fixed layout is that elements look consistent, no matter the size of the display. The disadvantage of fixed layouts is that they may overflow a small display. Fluid layouts set widths as percentage values so that the layout will resize as the browser window resizes. Try a fluid layout now. Set both divs to a width of 35%, and set the container to a width of 100%.
  9. What happens if you resize your browser window after setting the divs to the percentage setting in the previous step?
  10. In the html source code for this page, copy testDiv1 and paste the copy below testDiv2. Change the id for this new div to testDiv3. Add a #testDiv3 section to the style rules. Now you have three columns to play with.
  11. Try to make a three column display with all three columns equally sized.
  12. Try to make a three column display with the far left column about 25% of the page width, and the other two splitting the remaining 75%. Try this as both a fixed and a fluid layout.
